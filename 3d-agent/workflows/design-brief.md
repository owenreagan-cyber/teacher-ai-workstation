# Design Brief Workflow

## Purpose

Turn an idea into a clear design brief before any CAD is generated.

## Inputs needed

- Use mode
- Problem to solve
- Required dimensions in mm
- Material and color preferences
- Printer/profile if known
- Safety and IP/reference notes

## Steps

1. Ask the use mode: personal, classroom, business/commercial, repair/replacement, or reference-only.
2. Ask what the object must do.
3. Ask for target dimensions and constraints.
4. Identify printability risks.
5. Produce a concise design brief.

## Output format

- Design name
- Use mode
- User/problem
- Dimensions
- Materials/colors
- Constraints
- Printability notes
- Open questions

## Footer

- Use mode: personal / classroom / business-commercial / repair-replacement / reference-only
- Printability notes
- Safety notes
- IP/reference notes
- Human review recommended
- Human operator final decision
