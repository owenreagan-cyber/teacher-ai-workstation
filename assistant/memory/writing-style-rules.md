## Purpose

Track approved writing style rules for Owen's Chief of Staff.

## Current Confidence

## Confidence Update Process

- Confidence starts at Low.
- Confidence must be manually updated by Owen after reviewing real outputs.
- The CLI cannot promote writing-style confidence automatically.
- Each tier change should be recorded in `assistant/memory/memory-log.md`.
- A confidence upgrade should explain:
  - date
  - what samples or outputs triggered the change
  - what Owen confirmed
  - whether the change applies to email, parent communication, student-facing writing, internal planning, or all writing

## Style Precedence Rule

When this file is explicitly included with `--include-writing-style-memory` or `--include-memory`, `writing-style-rules.md` takes precedence over `assistant/writing-style.md` for style decisions because it represents reviewed, personalized memory.

However, `writing-style-rules.md` cannot override:

- safety rules
- permission rules
- sensitivity rules
- source-verification rules
- the current user instruction

## General Style Rules

- Be clear, practical, and direct.
- Prefer beginner-friendly steps.
- Use structured headings for complex tasks.
- Avoid overclaiming.
- Ask for missing context when needed.
- Use "Verify before use" for factual/curricular/technical claims.
