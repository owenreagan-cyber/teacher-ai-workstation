# 3D Printing Coordination Workflow

## Purpose

Coordinate 3D printing notes and routing only. This does not build the 3D Design Agent yet.

The Chief of Staff can route 3D design, business, and product tasks to a future standalone 3D Design Agent.

## Inputs needed

- Request summary
- Personal, classroom, or business mode
- Desired object or product
- Constraints
- Available files or notes

## What belongs to the future 3D agent

- Design files
- Filament specs
- Printer/slicer settings
- OpenSCAD/CadQuery code
- Product pricing
- Print-failure diagnosis
- Prototype logs
- Product listing drafts
- Copyright/IP checks
- Classroom fidgets/prizes
- Business product ideas

## What the Chief of Staff can handle now

- Project notes
- Vendor research notes
- Scheduling
- Priority planning
- Collecting design ideas
- Deciding whether a task should be routed to the future 3D agent

## Steps

1. Classify the request.
2. Check personal/classroom/business mode.
3. Gather required inputs.
4. Identify safety/IP concerns.
5. Decide whether the task belongs to the future 3D agent.
6. Prepare a structured design brief if handoff is needed.

## Future handoff protocol

- Classify request
- Check personal/classroom/business mode
- Gather required inputs
- Identify safety/IP concerns
- Pass structured design brief to 3D agent
- Receive summary back into project memory

## Output format

- Request class
- Mode
- Needed inputs
- Safety/IP notes
- Chief of Staff next action
- Future 3D agent handoff brief

## Safety notes

- Do not generate manufacturing instructions for unsafe objects.
- Do not assume copyright/IP clearance.
- Do not build or connect a 3D agent in Phase 1A.

## Required footer

- Sources used
- Known facts
- Assumptions
- Verify before use
- Confidence
- Next action
