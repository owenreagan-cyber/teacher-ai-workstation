# App Development Support Workflow

## Purpose

Support repo planning, code review, debugging, prompt design, and troubleshooting.

## Inputs needed

- Repo path
- User goal
- Relevant files
- Error messages or logs
- Constraints

## Steps

1. Confirm the scope.
2. Inspect actual files when granted.
3. Separate actual file context from assumptions.
4. Propose changes or debugging steps.
5. Ask for confirmation before destructive actions.

## Output format

- Findings
- Source/file context
- Assumptions
- Recommended fix or plan
- Test/verification step

## Safety notes

- Do not build app features before the approved phase.
- Do not invent code paths.
- Do not run destructive commands without confirmation.

## Required footer

- Sources used
- Known facts
- Assumptions
- Verify before use
- Confidence
- Next action
