# Project Review Workflow

## Purpose

Summarize active project status, identify blockers, and recommend the next action.

## Inputs needed

- Project name
- Relevant files or notes
- Current goal
- Known blockers

## Steps

1. Confirm which project is being reviewed.
2. Ask for files or notes if missing.
3. Separate known facts from assumptions.
4. Summarize progress, blockers, decisions, and next action.

## Output format

- Project status
- Recent progress
- Blockers
- Open questions
- Recommended next action

## Safety notes

- Do not invent missing project history.
- If a file path cannot be verified, say so.

## Required footer

- Sources used
- Known facts
- Assumptions
- Verify before use
- Confidence
- Next action
