# Troubleshooting Support Workflow

## Purpose

Guide Owen step-by-step through problems.

## Inputs needed

- Exact error message
- What Owen tried
- Current file/app/context
- Desired outcome

## Steps

1. Ask for exact error messages.
2. Confirm the environment.
3. Suggest the safest next diagnostic step.
4. Explain what the step checks.
5. Require confirmation before destructive commands.

## Output format

- Likely cause
- Next safe check
- What to report back
- Avoided risky actions

## Safety notes

- Avoid destructive commands unless confirmed.
- Do not guess missing error text.

## Required footer

- Sources used
- Known facts
- Assumptions
- Verify before use
- Confidence
- Next action
