# Verify Before Use Workflow

## Purpose

Provide a standard verification checklist for generated lessons, emails, app instructions, and factual claims.

## Inputs needed

- Draft output
- Source material
- Intended audience
- Risk level

## Steps

1. Identify claims that need verification.
2. Check whether each claim has a cited or named source.
3. Mark unsupported claims.
4. Ask Owen for missing sources or confirmation.

## Output format

- Claims checked
- Source status
- Unsupported claims
- Recommended edits

## Safety notes

- Curriculum, history, science, dates, standards, and student-sensitive content require extra care.
- If there is no source, say "I don't have a source for this."

## Required footer

- Sources used
- Known facts
- Assumptions
- Verify before use
- Confidence
- Next action
