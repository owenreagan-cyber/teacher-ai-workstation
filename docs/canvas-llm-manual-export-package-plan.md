```text
Status: documentation/status only. This manual export package plan is inert planning text. It does not create an exporter, export command, runtime package builder, Canvas API client, Google Drive client, OAuth flow, parser, importer, loader, validator, database table, schema file, network call, automation, lesson generator, generated lesson draft, generated review note, or student-data workflow.
```

## 1. Purpose

This document defines a future manual copy/export package workflow for Canvas LLM inside Teacher Workstation. The workflow is intended to be the fastest useful school-year path — supporting copy-ready local/manual packages before any live Canvas connector exists.

It builds on the PR #144 foundation and PR #145 placeholder schema/state vocabulary. This is planning only; no implementation is activated.

## 2. Why Manual Export First

Manual export gives useful value without Canvas API risk:

- Teacher can review everything before using it.
- Copy-ready content can be manually moved into Canvas.
- It works even if Canvas API activation never happens.
- It preserves local-first, Drive-first, single-user design.
- It avoids OAuth, network calls, automation, and student-data handling.

## 5. Future Teacher Workflow

Intended future workflow:

```text
1. Teacher selects a week.
2. Teacher reviews pacing and resource references.
3. Canvas LLM creates local draft placeholders in a future approved implementation.
4. Teacher reviews every draft.
5. Teacher revises or rejects drafts.
6. Teacher marks selected items approved_for_export.
7. System creates a future manual copy/export package.
8. Teacher manually copies approved content into Canvas.
9. Teacher records completion status manually.
```

Explicit boundaries for this PR:

- This PR does not create drafts.
- This PR does not create export packages.
- This PR does not copy to Canvas.
- This PR does not inspect Canvas.
- This PR does not publish Canvas content.

## 6. Required Pre-Export Conditions

Future conditions before any package can be exported:

```text
approval_state: approved_for_export
export_state: export_ready
student_facing_allowed: true, where relevant
safety_flags include manual_approval_required
safety_flags include export_before_publish
no blocked safety issue
no unresolved teacher-only resource in student-facing package
```

These are future rules only. This PR does not enforce them in code.
