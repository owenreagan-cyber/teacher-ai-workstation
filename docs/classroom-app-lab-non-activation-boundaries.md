## Purpose

Define hard boundaries for the Classroom App Lab Prototype Rescue foundation and all future Classroom App Lab work until explicit implementation approval.

## Blocked Now (Foundation and Default)

| Capability | Status |
| --- | --- |
| Codebase zip upload | blocked |
| Zip extraction / unzip | blocked |
| App repo ingestion | blocked |
| App code parsing | blocked |
| App execution / testing | blocked |
| LLM/API analysis loops | blocked |
| Automatic Cursor execution from analysis | blocked |
| Automatic code modification | blocked |
| Student data handling | blocked |
| Real classroom roster data | blocked |
| Network / API / OAuth calls | blocked |
| Document/folder scanning | blocked |
| OCR / embeddings / vector DB / RAG | blocked |
| Background jobs / schedulers | blocked |

## Allowed Now (Foundation)

- Repo-local documentation and prototype rescue planning concepts
- Metadata-only inventory of unfinished classroom app categories
- Deterministic status script reporting planning boundaries
- Chief of Staff read-only status via `--classroom-app-lab-status`
- Cross-links to Widget and Shortcut Builder, Lovable (G1), and Chief of Staff intake

## Relationship to Other Programs

| Program | Boundary |
| --- | --- |
| Widget and Shortcut Builder (F1) | App-launch shortcut concepts — no install |
| Lovable Classroom App Builder (G1) | External builder — no API connection |
| Chief of Staff intake | Capture app ideas — no auto-analysis |
| Curriculum Builder | Separate track — no curriculum ingestion |
| Integration Layer (G) | Last-stage — no live connectors |

## Future Stages (Approval-Gated)

| Stage | Requires |
| --- | --- |
| CAL2 Prototype inventory manifest | Explicit approval mission |
| CAL3 Zip intake dry-run | Explicit approval + safety review |
| CAL4 Code analysis assist | Explicit approval + no auto-modify |
| CAL5 App test harness | Explicit approval per app |
| CAL6 Rescue workflow automation | Blocked — separate mission |

The foundation must not blur into any future stage.
