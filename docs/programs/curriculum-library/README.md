# Curriculum Library / Teacher Knowledge Vault Program

```text
Status: PARTIAL local-first planning/prototype track
Classification: metadata/reference-based
```

Curriculum Library and Teacher Knowledge Vault organize curriculum knowledge through local-first metadata and references. Large curriculum files remain in Google Drive, NAS, iCloud, or local folders.

The repo/app may store metadata, references, hashes, tags, review state, source links, relationships, and safety status. It must not copy, ingest, OCR, embed, summarize, or index real curriculum content unless separately approved.

Reality audit: `docs/master-plan/build-state-checklist.md` classifies this track as **PARTIAL** because repo-backed docs/status/fake fixtures and bounded prototype catalogs exist, while general scanning, ingestion, connectors, and production/canonical writes remain blocked.
