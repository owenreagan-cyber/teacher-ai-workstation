# Teacher Workflow Safe-Output Examples

## Purpose

This document adds safe example outputs for Teacher Planning, Lesson Review, Review Notes, and Document Indexing planning status commands after teacher workflow command detail summary. It makes expected output easy to recognize without changing command behavior.

This PR adds Teacher Workflow safe-output examples only. This pass is about preserving all existing commands, preserving command behavior, and preserving PASS/WARN/FAIL semantics while enforcing no command removals, no command renames, and no check removals.

## Current Status

```text
Current status: teacher workflow safe-output examples in progress.
```

Safe-output example rules are documented for local status commands only.

## Why Safe Output Examples Exist

After teacher workflow command detail summary, command detail docs explained purpose and boundaries but users still needed recognizable output shapes. These examples show what safe planning/status output should look like without implying real lesson generation, real review notes, document scanning, file indexing, or student data handling exists.

## Safe Output Example Rules

```text
safe examples must be documentation only
safe examples must not be generated by command behavior changes
safe examples must not include real lesson content
safe examples must not include real review notes
safe examples must not include student data
safe examples must not imply document scanning exists
safe examples must not imply file indexing exists
safe examples must not imply OCR, embeddings, or vector database behavior exists
safe examples should make PASS/WARN/FAIL semantics easy to recognize
safe examples should make planning-only boundaries obvious
```
