## Purpose

This brief gives future visual work a consistent teacher-centered direction while keeping implementation blocked until explicitly approved.

## Teacher Workflow Fit

The workstation should feel like a calm classroom command desk: fast status at a glance, lightweight mode awareness, and a sense that prep work, classroom utilities, and review materials belong to one coherent local system.

The visual language should support:

- Morning setup and decision-making.
- In-class transitions where attention and timing matter.
- Planning review after a school day.
- Local-first confidence, with clear signals when a lane is parked or blocked.

## Visual Goals

- Calm utility first: readable, grounded, and not decorative at the expense of scanning.
- Classroom energy second: color and concept names may be warm, playful, or seasonal when they still preserve focus.
- Status clarity: planning, blocked, approved, and runtime states should look different in future designs.
- Teacher agency: Owen should be able to choose modes, not have the system silently mutate the desktop.

## Classroom Utility Goals

Future classroom tools should share a clear visual language: large timer states when used live, quiet planning panels when used after class, and consistent blocked-state labels when a tool is only planned.

## Safe Example Policy

Examples in this lane are fake/local illustrations. They may include concept names, mood labels, and blocked action lists. They must not include real Mac paths, real shortcut payloads, widget code, AppleScript, LaunchAgents, API keys, OAuth settings, student data, or real curriculum content.

## Owen Approval Needed

Owen must explicitly approve any mission that changes the Mac, installs a widget, installs or runs a shortcut, creates an executable visual shell, activates app runtime behavior, or connects to an external service.
