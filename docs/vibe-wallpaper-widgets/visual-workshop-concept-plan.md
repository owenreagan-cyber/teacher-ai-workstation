## Purpose

This plan describes future teacher dashboard, classroom utility, and 3D workshop visual direction without building executable UI.

## Concept Direction

- Teacher dashboard: compact status cockpit with clear blocked/approved/runtime labels.
- Classroom utility visual language: simple controls, visible state, readable distance-friendly displays when runtime is separately approved.
- 3D workshop: maker-table organization, zones for idea, model, verify, print, and reflect. This is direction only; no CAD, slicing, or printing.
- Future Mac/local shell: a possible command-center surface, blocked until an explicit runtime mission.

## Safe Static Wireframe Description

The future visual workshop may be described as five zones: `Plan`, `Queue`, `Build`, `Verify`, and `Show`. In this lane those are words in a document and fake JSON only, not a page, component, app, or executable shell.

## Safety Boundaries

This plan does not approve React/Vue/Svelte components, HTML pages, canvas scenes, Three.js, app wrappers, local model calls, app launches, or desktop modifications.

## Future Gate

A future visual shell mission must state whether it is a non-runtime design artifact, a local runtime app, or a Mac-integrated surface. Runtime or Mac-integrated work requires Owen approval and must keep student data, real curriculum ingestion, external integrations, and production writes blocked unless separately approved.
