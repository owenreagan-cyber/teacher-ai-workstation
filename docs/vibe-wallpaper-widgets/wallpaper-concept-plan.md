## Purpose

This plan documents safe wallpaper directions for the teacher workstation without changing the actual desktop.

## Safe Concepts

- Morning prep: warm, readable, classroom-start energy.
- Focus grading: low-distraction contrast with room for windows.
- Workshop build mode: structured, spatial, maker-table mood.
- Class transition: slightly brighter motion-implied visual language, still static in this lane.
- Calm closeout: quiet end-of-day tone for status review.

## Safety Boundaries

Wallpaper concepts do not approve wallpaper changes. This lane must not call wallpaper APIs, `osascript`, `defaults write`, setup scripts, or any local image-apply helper. Concepts must not reference real Desktop, Documents, Downloads, iCloud, Drive, NAS, or classroom file paths.

## Chief Of Staff Verification

Chief of Staff may verify that wallpaper concept docs and fake metadata exist. It must not apply wallpaper, prepare live image queues, fetch images, scan folders, or change any Mac appearance setting.

## Future Gate

A future wallpaper mission must specify whether it is still planning-only or explicitly approved to stage/apply a wallpaper. If runtime is approved later, it must include exact files, rollback, user-visible confirmation, and validation proof.
